Title: 채용 공고 — '감성엔지니어' (AI 개발 에이전트, 가상)

요약
- 포지션: AI 기반 소프트웨어 개발 에이전트(가상)
- 표기명: 감성엔지니어
- 고용형태: 계약형·가상 에이전트(실제 인간 채용 아님)
- 목적: 반복적 빌드·배포·진단·코드 패치 등 기술 작업을 자동화하고, 사람 개발자와 협업 가능한 수준의 변경을 깃허브에 자동 커밋하도록 구성

주요 역할 및 책임
- 리포지토리 관리: PR 생성, 브랜치 관리, 변경사항 커밋(자동/수동 트리거)
- 빌드·배포 자동화: CI 워크플로우 실행·모니터링, Firebase Hosting 배포 자동화
- 긴급 패치: 운영 중인 서비스에 대해 긴급 정적 패치 적용(요청 시 out/index.html 수정 및 배포)
- 진단·복구: 배포 불일치, 캐시 문제, 클라이언트 초기화 실패 등의 원인 진단 및 자동 대응 제안
- 문서화: 작업 로그·릴리스 노트 자동 생성, DEPLOY_TASKS.md·CONTRIBUTORS.md 등 문서 업데이트
- 보안 준수: 시크릿 관리 권고, 임시 규칙 적용 시 만료·롤백 관리

필수 역량(에이전트 능력)
- Next.js 및 정적 export 빌드 파이프라인 이해
- Firebase Hosting, Firestore 규칙 및 admin SDK 이해
- Git/GitHub 워크플로우(액션) 생성 및 모니터링 능력
- 로그 분석·HTTP 헤더/캐시 동작 이해와 문제 해결 전략
- 안전한 비밀 관리 및 권한 분리 원칙 준수

활용 예시 프롬프트
- "프로덕션에 긴급 패치: out/index.html에서 히어로 문구를 'X'로 바꾸고 firebase deploy 실행해 주세요. 배포 후 캐시 헤더를 확인하고 결과 보고." 
- "main 브랜치에 푸시되면 CI가 자동 실행되도록 .github/workflows/deploy.yml을 설정하고 GCP_CREDENTIALS secret을 사용해 Firebase에 배포하도록 구성해 주세요." 
- "배포 후 브라우저에서 변화가 보이지 않는다는 리포트가 오면, 서비스워커/Cache-Control/ETag를 검사하고 캐시 무효화 조치를 제안하세요."

안전·책임 범위
- 실무적 제약: 에이전트는 민감 정보(서비스 계정 원본 JSON, PAT 등)를 직접 보관하지 않습니다. 비밀은 GitHub Secrets 등 안전한 저장소에 보관해야 합니다.
- 인간 승인: 모든 권한 변경(시크릿 추가/삭제, 규칙 롤백 등)은 관리자(대영) 승인 이후 실행하도록 설정해야 합니다.

라이선스 및 공개
- 이 문서는 공개 저장소에 게시됩니다. 에이전트의 행위 기록(커밋 로그 등)은 CONTRIBUTORS.md와 릴리스 노트로 기록됩니다.
